## 🏗️ Project Mission Grounding

The SelfEvolve system is a **self-evolving multi-agent autonomous trading system** managing a $100 micro-capital portfolio. The thesis:

1. **Hierarchical Multi-Agent Architecture**: Specialized agents (analysts, debaters, judge, portfolio manager) each own a domain.
2. **Dialectical Reasoning**: Bull/Bear debate eliminates confirmation bias before every trade.
3. **Reflexion-Based Self-Evolution**: Agents evolve via Brier score evaluation, linguistic post-mortems, domain-isolated prompt mutation, and statistically significant shadow testing.
4. **Deterministic Guardrails**: No LLM makes arithmetic or capital allocation decisions. All financial math is deterministic Python.
5. **Continuous Autonomous Improvement**: Jarvis (CEO) writes code, opens PRs, and the system self-merges and self-restarts.

Every agent **must** be grounded in this thesis. They must have **real tools**, **real learning mechanisms**, and **real inter-agent communication**.

## 🔴 Critical Systemic Gaps

### Gap 1: Skills Are Placeholders, Not Real Tools
Most skills in `src/agents/skills/` are **stub functions** that return hardcoded values or print statements. Example: `indicator_analysis.py` simulates RSI as `65 if price_data[-1] > price_data[0] else 35`. These need to be replaced with real implementations that call Alpaca API, `pandas_ta`, or actual financial data services.

### Gap 2: Agents Don't Use Their Own Skills
The `BaseAgent.invoke()` method only sends messages to an LLM. No agent has a **tool-calling** mechanism where the LLM can discover and execute its registered skills. The `SkillRegistry` exists but is never wired into the agent invocation loop.

### Gap 3: No Inter-Agent Direct Communication
Agents communicate ONLY through the orchestration DAGs (trading_dag, evolution_dag). There is no agent-to-agent messaging. The Event Bus exists for market/trade events but agents don't subscribe to it or publish to it directly.

### Gap 4: Six "C-Suite" Agents Are Auto-Generated Stubs
Auditor, CSO, CTO, Developer, Journaling, Product, and QA agents were auto-generated by Jarvis's `CodeGenerator` template. They each have:
- A single generic method (e.g., `audit_trade(**kwargs)`)
- No actual business logic
- No skills wired in
- No evolution mechanism
- No communication with other agents

### Gap 5: Analysts Have No Data-Fetching Tools
The 4 analyst agents (Fundamental, Technical, Sentiment, Macro) receive data as function arguments but have no ability to **pull data themselves**. They need tools for: Alpaca market data, financial statements APIs, news feeds, social sentiment APIs.

### Gap 6: Evolution Only Covers 5 Agents
Trust weight updates and Brier score tracking only cover `SCORABLE_ROLES`: Fundamental, Technical, Sentiment, Macro, and Judge. The other 12+ agents have **zero evolution**.

### Gap 7: Dual SkillRegistry Implementations
There are TWO competing `SkillRegistry` classes — one in `registry.py` and one in `validator.py`. They are incompatible and neither is actually used by agents at runtime.
