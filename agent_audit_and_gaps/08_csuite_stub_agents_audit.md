## Pattern: The Template Problem

Every agent was generated from this template in `code_generation.py`:

```python
class {ClassName}(BaseAgent):
    def __init__(self, llm, trust_weight=1.0):
        identity = AgentIdentity(...)
        super().__init__(identity, llm, trust_weight)

    async def execute_task(self, **kwargs) -> dict:
        message = f"Execute the following task:\\n{kwargs}"
        return await self.invoke(message)

    def _safe_default(self, error):
        return {"content": f"Error: {error}", "status": "error"}
```

This means:
- ❌ No domain-specific methods
- ❌ No Pydantic output schemas
- ❌ No deterministic logic (everything sent to LLM)
- ❌ No tool integration
- ❌ No structured error handling

## Cross-Cutting Gaps for ALL C-Suite Agents

### Gap CS-1: No Evolution Mechanism (🔴 CRITICAL)
None of these agents are in SCORABLE_ROLES. They never get Brier scores, trust updates, post-mortems, or prompt evolution. They are static forever.

### Gap CS-2: No Inter-Agent Communication (🔴 CRITICAL)
These agents should be talking to each other and to the main trading agents:
- Auditor should flag compliance issues to Judge
- QA should validate outputs from all agents
- CTO should monitor system health and alert Jarvis

- Developer should receive bug reports from QA

### Gap CS-3: No Event Bus Subscriptions (🔴 HIGH)
None subscribe to the Event Bus. The Journaling Agent especially should subscribe to `TRADE_EVENTS` to auto-document trades.

### Gap CS-4: Redundancy with Evolution Module (🟡 MEDIUM)
The `evolution/` directory has real implementations:
- `engineer_agent.py` → Does what Developer Agent should do
- `bug_scanner.py` → Does what QA Agent's bug detection should do
- `bug_worker.py` → Does what Developer Agent's fix proposals should do

These should be unified or the stub agents should be removed.

### Gap CS-5: No Structured Output (🟡 MEDIUM)
All agents return free-form dicts. Should use Pydantic models for their outputs.
