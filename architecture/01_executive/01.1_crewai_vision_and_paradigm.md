# CrewAI Vision and Paradigm

## Architectural Shift
Transitioning from deterministic state machines to CrewAI represents a paradigm shift toward role-playing, conversational autonomy. CrewAI models the trading firm as an explicit team of specialized professionals rather than a rigid graph of nodes.

## Professional Implementation
- **Backstory Engineering**: In a professional CrewAI setup, agents are not merely prompted; they are given deep, multi-paragraph backstories that define their historical market biases, preferred sectors, and exact mathematical boundaries.
- **Hierarchical Processes**: The system will utilize CrewAI's `HierarchicalProcess` where a Manager Agent acts as the Portfolio Manager, dynamically delegating tasks to Sub-Crews rather than following a static sequential chain.

## Evolution Capability
- The vision requires that the Manager Agent evaluates the performance of its Sub-Crews at the end of each epoch.
- If a Sub-Crew consistently underperforms (e.g., the Technical Agent misreads volatility), the Manager Agent programmatically updates the Technical Agent's `backstory` and `role` to force a more conservative approach in subsequent executions.
