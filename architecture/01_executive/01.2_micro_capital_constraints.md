# Micro-Capital Constraints

## The $100 Sandbox
Building a system designed to operate on a $100 micro-capital account forces hyper-efficiency. Every basis point matters.

## Professional Implementation
- **Fractional Shares**: The system must utilize brokerages (like Alpaca) that support fractional trading down to 4 decimal places.
- **Zero-Commission Mandate**: The strategy is restricted from trading assets or using routing networks that incur flat ticket fees, which would instantly erode the capital base.

## Evolutionary Mechanism
If the system detects that bid-ask spreads on penny stocks are consuming 5% of the $100 capital per trade, it autonomously evolves its asset selection criteria to only include high-liquidity, large-cap stocks where fractional trading slippage is statistically negligible.

---

## 🛑 VC Challenge & Investor Critique
**The Scale-Up Fallacy:** Investors will challenge the assumption that a strategy optimized for $100 will scale to $1,000,000. Fractional shares suffer from poor execution routing (often internalized by the broker rather than hitting the lit exchange). Furthermore, API token costs (GPT-4) will obliterate a $100 account. If you spend $2 a day on API calls, you need a 500% annualized return just to break even.

### Critical User Journeys (CUJs)
1. **The API Token Bleed:** The CrewAI system enters a Debate loop, consuming 500k tokens. The trade makes $0.50, but the API bill is $2.50. *Result: Negative Net PnL, death by operational cost.*
2. **Fractional Liquidity Trap:** The LLM signals a sell on $50 of a highly volatile micro-cap. The fractional execution engine at the broker delays the fill by 2 minutes to batch it. *Result: 15% slippage on exit.*

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Token-Aware PnL Flow**: The CrewAI `Task` must be wrapped in a `CostTrackingFlow`. Before any execution, a `NetExpectedValueNode` calculates: `(Expected Profit) - (Estimated Token Cost of the Agent Flow)`. If EV is negative, the trade is aborted before the LLM even begins heavy reasoning.
2. **Small-Model Triage**: Upgrade the architecture to use Local LLMs (Llama 3 8B via Groq) or lightweight models (Claude Haiku) for 90% of data parsing, reserving premium models (GPT-4o) *only* for the final `Judge Agent` decision, drastically reducing operational drag.
