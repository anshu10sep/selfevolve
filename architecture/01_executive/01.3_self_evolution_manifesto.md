# Self-Evolution Manifesto

## The Goal
The system should require zero code changes from the human operator over a 12-month period, adapting to market regimes autonomously.

## Professional Implementation
- **Prompt as Code**: Treating agent system prompts as living variables stored in a database, not hardcoded strings.
- **The Reflexion Cycle**: A nightly batch process where the system scores its own trades and rewrites its own instructions based on empirical success or failure.

## Evolutionary Mechanism
If a new macroeconomic regime begins (e.g., stagflation), the system observes a sudden drop in its win rate. The Reflexion cycle diagnoses the regime change and rewrites the core backstories of the Fundamental and Macro agents to penalize growth stocks and favor commodities.

---

## 🛑 VC Challenge & Investor Critique
**The Overfitting & Catastrophic Forgetting Risk:** A VC will immediately recognize that "Self-Evolution" often leads to "Catastrophic Forgetting." If the system rewrites its prompts to survive a 2-week bear market, it will forget how to trade the subsequent bull market. Furthermore, giving an LLM write-access to its own core logic is a massive security/stability risk.

### Critical User Journeys (CUJs)
1. **The Death Spiral:** The system hits a bad streak due to a black swan event. The Meta-Review Crew panics and rewrites all agent prompts to be hyper-conservative ("Never buy anything"). *Result: The system becomes paralyzed and stops trading entirely.*
2. **Prompt Injection Loop:** The LLM misinterprets its own feedback, inserting a contradictory instruction into the database (e.g., "Always buy tech, but never buy tech"). *Result: CrewAI loops infinitely trying to resolve the paradox.*

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Immutable Core & Mutable Strategies**: The database schema for agents must be split into `Core_Directives` (Immutable: Risk limits, core mission) and `Current_Hypothesis` (Mutable). The Meta-Review Crew is *only* allowed to edit the hypothesis.
2. **Version Control & A/B Testing**: Evolution must be scientifically validated. When the Meta-Review Crew generates a new prompt (v2), it does NOT replace v1. Instead, it spins up a *Shadow Crew* using v2 that paper-trades alongside the live v1 Crew. Only if the Shadow Crew achieves a statistically significant higher Sharpe ratio over a 7-day rolling window does v2 get promoted to Production.
