# Cash Account Monitoring Crew

## Role Context
The $100 constraint forces operation in a cash account, making the distinction between "settled" and "unsettled" cash the most critical data point in the entire system.

## Professional Implementation
- **Dedicated Sub-Crew**: The `Cash Management Crew` runs a background sequential process asynchronously.
- **Tools**: Utilizes `AlpacaAccountQueryTool` to parse the specific `cash_withdrawable` and `cash_pending` endpoints.
- **Output Validation**: Forces output into a strict Pydantic model `CashStateModel` which is injected into the context of the Main Trading Crew.

## Evolutionary Mechanism
The Cash Monitoring agents track their own latency and API rate-limit errors. If polling Alpaca every 5 minutes causes 429 errors, the agent autonomously evolves its polling schedule logic to utilize WebSockets or back-off multipliers, ensuring seamless cash monitoring.
