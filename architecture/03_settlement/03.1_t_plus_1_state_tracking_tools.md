# T+1 State Tracking Tools

## Custom Tool Architecture in CrewAI
CrewAI agents interface with systems via custom tools. To manage T+1 settlement accurately, we construct deterministic tracking tools that agents *must* use.

## Professional Implementation
- **Tool**: `LedgerTrackerTool(BaseTool)`
- **Functionality**: Integrates with a local SQLite/PostgreSQL database to track the exact minute a trade is closed, calculating the exact clearing date based on market holidays and weekends.
- **Integration**: The `Risk Manager Agent` cannot approve a trade without attaching a `LedgerTrackerTool` output showing sufficient cleared funds.

## Evolutionary Mechanism
The tool tracks its own historical accuracy. If the broker changes settlement logic (e.g., the transition from T+2 to T+1), the agent observing the clearing discrepancies uses a `ToolUpdateAction` to rewrite the Python logic inside the `LedgerTrackerTool` dynamically, fixing its own calculation offset.
