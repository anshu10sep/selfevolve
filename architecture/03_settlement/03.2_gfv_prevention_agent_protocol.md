# GFV Prevention Agent Protocol

## Role Context
Good Faith Violations (GFV) are fatal to a micro-capital system. A strict protocol must intercept trades that would trigger a GFV.

## Professional Implementation
- **Agent**: `GFVAuditor`
- **Process Injection**: Executed as the final sequential step in the Trade Approval Crew. 
- **Logic**: It compares the `asset_purchase_date` of the currently held asset against the `settled_cash_date` of the funds used to buy it. If the funds are not yet cleared and a sell order is proposed, the Auditor explicitly issues a `REJECT_GFV_RISK` command.

## Evolutionary Mechanism
The Auditor actively monitors the broker's GFV warning endpoints. If the broker flags a near-miss, the Auditor agent triggers an immediate self-reflection, updating its system prompt to widen its safety margin (e.g., waiting an extra 2 hours after theoretical clearing time to execute).
