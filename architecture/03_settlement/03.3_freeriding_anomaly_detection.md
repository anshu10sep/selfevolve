# Freeriding Anomaly Detection

## Role Context
Freeriding occurs when a security is sold before sufficient funds have settled to cover the initial purchase. The system must proactively detect these anomalies.

## Professional Implementation
- **Tool**: `FreeridingDetectorTool`
- **Mechanics**: Instead of just checking cash balances, this tool cross-references the chronological order of executions across the broker's ledger and the internal state.
- **Crew Integration**: The `Auditor Agent` runs this tool asynchronously as a continuous background check.

## Evolutionary Mechanism
The system evolves its anomaly detection by learning from simulated paper-trading environments. When a synthetic freeriding violation is purposefully triggered during Phase 2 testing, the `Meta-Review Crew` analyzes the exact sequence of events, extracting the signature of the failure, and encoding it as a permanent rule within the `Auditor Agent`'s core memory.
