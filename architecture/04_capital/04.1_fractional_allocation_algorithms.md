# Fractional Allocation Algorithms

## Maximizing the $100 Base
When buying a $400 stock (e.g., MSFT) with a $100 account, you cannot buy whole shares. The system must natively understand and request fractional routing.

## Professional Implementation
- **Notional Trading Setup**: The Alpaca API supports `notional` (dollar-based) order execution. 
- **The Math Wrapper**: The LLM is instructed to output exact dollar amounts (e.g., "$12.50"). A deterministic Python tool intercepts this, checks if the target ticker supports fractional routing on Alpaca, and constructs the API payload: `{"symbol": "MSFT", "notional": "12.50", "side": "buy"}`.

## Evolutionary Mechanism
If an asset is delisted from fractional support (which happens during corporate actions), the Alpaca API throws an error. The Meta-Review Crew logs this error, autonomously adds the ticker to a `Fractional_Blacklist` in the database, and evolves the asset-selection agent to strictly cross-reference this blacklist before generating ideas.

## 🛑 VC Challenge & Investor Critique
**The Fractional Fill Delay & Market Impact:** An institutional quant will challenge the viability of a high-frequency strategy built on fractional shares. Fractional orders are typically not routed to lit exchanges immediately; brokers internalize them or batch them for end-of-day execution. This means if your agent predicts a 10-minute breakout, the fractional order might sit pending for 15 minutes, completely missing the move.

### Critical User Journeys (CUJs)
1. **The Stale Fill:** The agent identifies a breakout and submits a $10.00 fractional market order. The broker batches it. It fills 20 minutes later at the top of the candle, right before the reversal. *Result: The system buys the top due to structural fractional latency.*
2. **The Unsupported Asset Crash:** The agent tries to fractionally short a stock. Most brokers do not allow fractional shorting. *Result: API error, execution loop fails.*

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Execution Latency Profiling**: We must update the `OrderExecutionNode` within the CrewAI Flow to record `time_submitted` vs `time_filled`. If the delta exceeds 60 seconds (indicating batching delays), the system automatically categorizes the asset as "Slow Fill" and updates the Strategy Agent to only apply long-term swing logic to it, abandoning breakout strategies.
2. **Hard Constraints in Pydantic**: The system must enforce constraints at the schema level. The `ExecutionSchema` restricts `side` to `BUY` or `SELL`. Shorting is programmatically disabled at the Flow level for fractional orders, preventing the LLM from ever attempting an illegal fractional short.
