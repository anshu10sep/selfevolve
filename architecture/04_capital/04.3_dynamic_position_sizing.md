# Dynamic Position Sizing

## The Kelly Criterion in AI
Allocating a flat $10 per trade ignores the probability of success. The system must size positions dynamically based on trade conviction.

## Professional Implementation
- **Conviction Scoring**: The Debate Crew (Bull vs Bear) outputs a final `Confidence_Score` between 0.0 and 1.0.
- **Sizing Math**: The Judge Agent uses a modified Kelly Criterion formula. If confidence is 0.9, it allocates 15% of settled capital. If confidence is 0.6, it allocates only 5%.

## Evolutionary Mechanism
If the system realizes that "High Conviction" (0.9) trades actually lose 60% of the time due to LLM overconfidence hallucination, the Meta-Review Crew autonomously rewrites the Kelly multiplier equation in the database, halving the maximum allowable allocation to protect capital against LLM hubris.

---

## 🛑 VC Challenge & Investor Critique
**The LLM Calibration Problem:** VCs know that LLMs are notoriously poorly calibrated; they are confidently wrong. Using an LLM's raw `Confidence_Score` as the direct multiplier for a Kelly Criterion bet sizing algorithm is a recipe for disaster. The LLM will eventually output a 0.99 confidence on a terrible trade, and the algorithm will size it massively, blowing up the account.

### Critical User Journeys (CUJs)
1. **The Confident Hallucination:** The agent hallucinates a fake earnings report beat, sets confidence to 1.0, and allocates 30% of the account. *Result: Massive, unrecoverable drawdown.*
2. **The Tiny Bet Bleed:** The agent becomes hyper-conservative due to previous losses, consistently outputting 0.4 confidence. The algorithm allocates $2 per trade. The fractional share slippage and API token costs consume the entire $2. *Result: Death by a thousand cuts.*

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Decouple Conviction from Size**: The LLM's `Confidence_Score` should *never* directly dictate position sizing. The architecture must adopt **Volatility-Targeted Sizing (Risk Parity)**. 
2. **Deterministic Sizing Node**: We introduce a `PositionSizingNode` in the CrewAI Flow. The LLM only provides the direction (Long/Short) and the underlying thesis. The `PositionSizingNode` looks at the asset's 14-day Average True Range (ATR). It calculates position size such that a 1-ATR move equals exactly 1% of the total account equity. This removes the danger of LLM overconfidence entirely, relying on pure mathematical volatility for risk management.
