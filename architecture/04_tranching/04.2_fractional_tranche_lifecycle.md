# Fractional Tranche Lifecycle

## Lifecycle Management
Managing $100 requires compartmentalizing capital into $5 or $10 fractional tranches.

## Professional Implementation
- **State Machine Integration**: While CrewAI handles reasoning, the tranche lifecycle is managed by an external deterministic wrapper that the Crew calls via custom tools.
- **States**: `AVAILABLE` -> `LOCKED_IN_TRADE` -> `SETTLING` -> `AVAILABLE`.
- **Crew Action**: The `Execution Agent` must specifically "checkout" an `AVAILABLE` tranche ID before routing an order to Alpaca.

## Evolutionary Mechanism
If the Crew observes that opportunities are being missed because too many tranches are stuck in the `SETTLING` state (e.g., tranches are too large, leaving no reserve capital), the `Meta-Review Crew` autonomously shrinks the default tranche size from $10 down to $5, increasing the total number of available "bullets" for the next session.
