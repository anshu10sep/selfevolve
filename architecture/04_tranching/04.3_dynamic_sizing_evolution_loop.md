# Dynamic Sizing Evolution Loop

## Continuous Optimization
The tranching logic cannot remain static as the $100 portfolio compounds into larger sums.

## Professional Implementation
- **Crew Task**: `OptimizeSizingTask`
- **Execution**: Runs every Friday post-market close.
- **Logic**: Evaluates the Sharpe ratio and Win/Loss metrics of the week's trades. It calculates the optimal Kelly Criterion based on empirical performance.

## Evolutionary Mechanism
The `Portfolio Manager Agent` receives the Kelly calculations from the `OptimizeSizingTask`. It uses a custom tool to edit the persistent YAML configuration files that dictate maximum tranche sizing. As the AI's win rate improves, it dynamically authorizes itself to deploy larger percentages of the portfolio, evolving from defensive micro-capital strategies to aggressive compounding strategies.
