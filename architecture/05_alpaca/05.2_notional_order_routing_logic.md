# Notional Order Routing Logic

## Handling Fractional Sizing
To maximize the $100 capital base, the system relies exclusively on fractional share purchasing via notional dollar amounts.

## Professional Implementation
- **Logic Layer**: The `AlpacaOrderSubmissionTool` intercepts any standard "quantity" inputs from the `Execution Agent` and forcefully translates them into the `notional` parameter.
- **Validation**: Ensures the notional value mathematically matches the cleared tranche allocation exactly before submission.

## Evolutionary Mechanism
The `Execution Agent` monitors slippage and partial fills resulting from fractional notional orders. If it detects that notional orders for highly illiquid penny stocks consistently result in poor fill prices, it evolves its routing logic to append specific `limit_price` parameters to notional orders, shifting from market-notional to limit-notional execution for specific asset classes.
