# WebSocket Stream Management

## Real-Time Ingestion
To react to the market, the system needs live data. REST API polling is too slow and burns rate limits. WebSockets are mandatory.

## Professional Implementation
- **Dedicated Daemon**: A Python `asyncio` script runs independently from the CrewAI orchestration. It maintains a persistent WebSocket connection to Alpaca's `sip` data feed.
- **Event Bus**: When significant volume or price spikes occur on watched tickers, the daemon pushes an event to a message queue (e.g., RabbitMQ or simple async queues). The CrewAI Orchestrator consumes these events and spawns targeted agent tasks.

## Evolutionary Mechanism
If the WebSocket drops frequently due to poor network handling, the daemon's internal error-logging triggers an evolution cycle. The Meta-Review Crew diagnoses the `TimeoutError` logs and autonomously rewrites the daemon's reconnection logic to implement more robust ping/pong keepalives.

## 🛑 VC Challenge & Investor Critique
**The LLM as a Developer Fallacy:** A VC with engineering experience will heavily scrutinize the "Evolutionary Mechanism" here. Allowing an LLM (the Meta-Review Crew) to autonomously rewrite the raw Python `asyncio` code for critical WebSocket daemon infrastructure is incredibly dangerous. LLMs are notoriously bad at writing complex, race-condition-free async code. The LLM is likely to introduce a fatal deadlock.

### Critical User Journeys (CUJs)
1. **The Poisoned Code Update:** The LLM "evolves" the WebSocket daemon by adding a blocking `requests.get()` inside the async event loop. *Result: The entire daemon freezes, the system goes blind, and live trades are left unmanaged.*
2. **The Reconnection Storm:** The WebSocket drops. The daemon reconnects immediately without backoff. Alpaca bans the IP for connection spamming.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Immutable Infrastructure Logic**: We explicitly ban the Meta-Review Crew from modifying infrastructure Python code (WebSockets, Database connections, Execution wrappers). Evolution is strictly confined to **Prompts, Agent Roles, and Strategy Parameters**. Infrastructure code is immutable and only updated via human-reviewed PRs.
2. **Decoupled Architecture**: Upgrade to a microservices pattern. The `Market Data Service` (WebSocket Daemon) runs in a separate Docker container. It broadcasts to a Redis Pub/Sub channel. The `CrewAI Orchestration Service` subscribes to this channel. If CrewAI crashes, the data stream remains perfectly healthy and uninterrupted.
