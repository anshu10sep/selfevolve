# API Rate Limit and Error Handling

## Network Resilience
Production trading systems cannot crash due to HTTP 429 (Rate Limit) or 503 (Service Unavailable) errors.

## Professional Implementation
- **Crew Wrapper**: A custom `RateLimitManager` sits between the Crew's tools and the Alpaca API. 
- **Task Delegation**: If an agent hits a rate limit, the `Manager Agent` pauses that specific sub-crew's execution thread and reallocates compute resources to an offline task (e.g., processing historical post-mortems) until the rate-limit window clears.

## Evolutionary Mechanism
The agents track API latency and failure distributions by time of day (e.g., API congestion right at the market open). Over time, the `Manager Agent` evolves the system's operational schedule, voluntarily delaying heavy data-scraping tasks until 09:35 AM to guarantee critical execution API calls have maximum priority and bandwidth at the 09:30 AM open.
