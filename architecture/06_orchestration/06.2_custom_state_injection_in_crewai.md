# Custom State Injection in CrewAI

## The Determinism Problem
Unlike LangGraph, CrewAI does not natively treat agents as nodes in a rigid state machine. To enforce strict T+1 capital tracking, we must inject an external state wrapper.

## Professional Implementation
- **State Injection Context**: We utilize CrewAI's `Task.context` to pass a strictly typed Pydantic object representing the current global portfolio state (available cash, pending settlements) to every task.
- **Before/After Callbacks**: Implementing `step_callback` and `task_callback` mechanisms to intercept an agent's output. Before the output is passed to the next agent, a deterministic Python function validates the math against the local SQLite database.

## Evolutionary Mechanism
If a callback consistently intercepts mathematical hallucinations (e.g., an agent trying to deploy 100% of capital when only 10% is settled), the callback logs the failure. The `Meta-Review Crew` reads these logs and updates the system prompt of the offending agent, adding rigid numerical examples to its backstory to prevent future hallucinations.

---

## 🛑 VC Challenge & Investor Critique
**The State Fragmentation Risk:** State injection via string templates or raw `Task.context` in CrewAI is prone to "context overflow." If the LLM's context window fills up with market data, it often "forgets" the injected state at the beginning of the prompt. VCs will challenge that relying on LLM attention mechanisms to respect $100 ledger balances is fundamentally unsafe.

### Critical User Journeys (CUJs)
1. **The Attention Drop:** The LLM reads a massive 10-K filing. The "Available Capital: $10" state injection was at the top of the prompt. The LLM's attention mechanism drops it, and it recommends a $5,000 trade based on the company's valuation. *Result: Trade rejected by Alpaca, execution fails.*
2. **The State Desync:** The `task_callback` successfully blocks a bad trade, but the CrewAI agent's internal memory *thinks* it executed the trade. For the rest of the session, the agent operates on a hallucinated portfolio state.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Pydantic State Injection via Flows**: We adopt the `CrewAI Flow` architecture. State is *not* just a string in the prompt; it is a live `FlowState(BaseModel)` object. 
2. **Hard Isolation of State**: The LLM is **never** asked to calculate or remember its available capital. The prompt is modified to simply output a `percentage_of_max_allocation` (e.g., 0.5 for 50%). The deterministic Flow mathematically translates that 0.5 into actual dollars based on the *true* database state right before hitting the API. The LLM is abstracted away from absolute dollar amounts.
