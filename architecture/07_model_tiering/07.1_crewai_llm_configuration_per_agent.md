# CrewAI LLM Configuration per Agent

## Strategic Resource Allocation
Running a full CrewAI instance exclusively on GPT-4o or Claude 3.5 Sonnet will destroy a $100 micro-capital portfolio through token fees alone.

## Professional Implementation
- **Agent-Level Mapping**: CrewAI allows explicit LLM bindings per agent.
- **Triage Tier**: Agents responsible for continuous scraping, parsing SEC filings, or formatting JSON strings are strictly bound to `gpt-4o-mini` or `claude-3-haiku`.
- **Reasoning Tier**: The `Judge Agent`, `Portfolio Manager`, and `Meta-Review Crew` are bound exclusively to premium models to prevent catastrophic risk-management failures.

## Evolutionary Mechanism
The system utilizes a custom `CostTrackerCallback` that monitors token expenditure vs realized profit per trade. If a specific "Triage" agent is consuming too much context without providing actionable insights, the `Meta-Review Crew` edits its system prompt to restrict its maximum output length or forces it to use a smaller extraction tool.

---

## 🛑 VC Challenge & Investor Critique
**The Context Window Cascading Failure:** While mapping cheaper models to triage tasks is standard, VCs will highlight the "Garbage In, Garbage Out" (GIGO) pipeline risk. If `gpt-4o-mini` hallucinates or misses a crucial nuance in an SEC 10-K due to its smaller parameter count, it passes that flawed summary up to the `Judge Agent` (GPT-4o). The premium model then makes a highly confident, catastrophic mistake based on cheap, flawed triage data.

### Critical User Journeys (CUJs)
1. **The Triage Hallucination:** A company files a complex 8-K detailing a toxic convertible debt spiral. The `gpt-4o-mini` agent summarizes it simply as "Debt restructuring announced." The GPT-4o Judge Agent interprets this as bullish and buys. *Result: Portfolio loses 20% due to a cheap model missing nuance.*
2. **Context Window Stuffing:** The Triage Agent passes 50,000 tokens of "summarized" data to the Judge Agent. The Judge Agent hits a $0.50 context charge just to read the summary.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Hybrid RAG instead of LLM Summarization**: Triage tasks should almost never use LLMs to "summarize" massive filings. Instead, we use deterministic Python to chunk the 10-K and embed it into a local vector database. The premium `Judge Agent` uses semantic search to pull *exact* quotes from the filing, completely bypassing the smaller model's hallucination risk.
2. **Confidence-Based Escalation**: If a smaller model *must* be used, it must output a `parsing_confidence_score`. If the score is low (e.g., encountering complex legalese), the Flow automatically escalates the specific paragraph to a premium model for translation, ensuring cost savings without sacrificing critical accuracy.
