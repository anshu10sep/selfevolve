# Hierarchical Crew Structures

## The Portfolio Management Paradigm
In a professional trading firm, junior analysts do not execute trades; they present research to a Portfolio Manager. CrewAI perfectly models this through Hierarchical processing.

## Professional Implementation
- **The Manager LLM**: Instantiating a highly capable `Manager Agent` (using Claude 3.5 Sonnet) that does not perform direct research.
- **Sub-Crews**: Deep research agents (Fundamental, Technical) operate as independent workers. The Manager delegates specific questions (e.g., "Analyze the 5-minute VWAP for $AAPL") rather than letting agents run freely.

## Evolutionary Mechanism
The `Manager Agent` retains an internal "trust score" for each of its Sub-Crews based on historical accuracy. If the Technical Analyst repeatedly recommends failed breakouts, the Manager evolves its delegation logic—automatically discounting the Technical Analyst's input and requesting deeper verification from the Macro Analyst before proceeding.

## 🛑 VC Challenge & Investor Critique
**The Single Point of Failure (SPOF):** VCs look for systemic fragility. If the entire architecture relies on a single `Manager Agent` LLM to delegate and synthesize everything, that LLM is a massive bottleneck. If the Manager LLM hallucinates, hits a context limit, or misinterprets a sub-agent's output, the entire system fails, regardless of how good the sub-agents are.

### Critical User Journeys (CUJs)
1. **The Information Bottleneck:** The Fundamental Agent outputs a brilliant, nuanced 500-word thesis on why a stock will crash. The Manager LLM summarizes it poorly in its internal monologue and ignores it, focusing on a weak technical signal instead. *Result: Loss of alpha due to Manager synthesis failure.*
2. **The Death Spiral of Trust:** The Manager Agent's "trust score" for the Technical Agent drops to 0 due to a bad streak. The Manager stops consulting technicals entirely. The market shifts to a purely technical regime. *Result: Portfolio gets decimated because the Manager refused to adapt.*

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Ensemble Voting over Dictatorship**: The Manager Agent should not be a single LLM deciding everything. We shift to an **Ensemble Flow Architecture**. The Sub-Crews (Technical, Fundamental, Macro) each output a structured `ConvictionScore` (-1.0 to 1.0). The "Manager" is replaced by a deterministic Python script that calculates a weighted average of these scores based on historical regime accuracy.
2. **Transparent Trust Decay**: The "trust score" evolution cannot be hidden in the Manager's latent space. It must be explicitly calculated by the `Meta-Review Crew` via statistical analysis of past trades (e.g., Brier score) and updated in the PostgreSQL database. The deterministic ensemble script reads these exact weights.
