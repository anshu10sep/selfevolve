# User Context Injection Tools

## Keeping the AI Aligned
An autonomous system must constantly reference the human operator's baseline constraints.

## Professional Implementation
- **Tool**: `UserContextFetchTool`
- **Mechanism**: The Manager Agent utilizes this tool at the start of every sequence to retrieve the global `UserContext` JSON object, which dictates the user's current risk appetite (e.g., "Aggressive" vs "Conservative") and specific blocked sectors (e.g., "Do not trade Crypto").

## Evolutionary Mechanism
The `UserContext` serves as the anchor. However, if the system's self-evolution logic detects that the user's requested "Aggressive" profile is mathematically incompatible with the current high-VIX market regime, it autonomously generates a `ContextOverrideProposal`. It pauses execution and alerts the human operator that continuing aggressively will likely result in liquidation, suggesting a temporary evolution to a "Conservative" profile.

**The Asynchronous Blocking Flaw:** An autonomous system pausing execution to ask a human for a `ContextOverrideProposal` during a high-VIX market crash defeats the entire purpose of automation. If the human operator is asleep or on a flight, the system is paralyzed exactly when it needs to act most decisively.

### Critical User Journeys (CUJs)
1. **The Sleep-Mode Crash:** VIX spikes to 40 at 3:00 AM. The system detects "Aggressive" is unsafe. It pauses to ask the user for permission to go "Conservative." The user is asleep. The portfolio is liquidated by 4:00 AM because the system refused to act without an override.
2. **Tool Failure Loop:** The `UserContextFetchTool` fails due to a database lock. The Manager Agent loops repeatedly trying to fetch the context, missing the entire trading day.

## đź› ď¸Ź Refinement & Architecture Update
To make this robust:
1. **State Injection via Flow, Not Tool**: The LLM should not waste a turn (and latency/tokens) calling a tool just to know its own rules. The `UserContext` is fetched by the Python `CrewAI Flow` at instantiation and injected directly into the prompt state before the LLM is ever invoked.
2. **Auto-Downgrade, Manual-Upgrade**: The system never pauses to ask for permission to reduce risk. If mathematical risk limits are breached, it *autonomously downgrades* the profile to "Conservative" immediately and sends an async notification. It only requires explicit human approval to *upgrade* risk (e.g., Conservative -> Aggressive). This ensures safety is strictly default-on.
