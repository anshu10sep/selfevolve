# EDGAR Parsing Tools for CrewAI

## Extracting Alpha from Filings
The Fundamental Analyst Agent cannot read unstructured HTML 10-K filings; it requires surgical data extraction.

## Professional Implementation
- **Custom Tool**: `SecEdgarXBRLParserTool`
- **Functionality**: Uses the SEC's XBRL API to extract exact, standardized numerical fields (e.g., TotalAssets, NetIncomeLoss) directly into Pydantic models. It skips the narrative boilerplate, drastically saving token costs.

## Evolutionary Mechanism
If the SEC updates XBRL tags or formatting, the parser will fail. The `Fundamental Analyst Agent`, upon receiving an execution error from the tool, enters a self-healing loop. It utilizes a separate `WebScrapingTool` to research the new SEC data structure, and dynamically rewrites the extraction logic within the `SecEdgarXBRLParserTool` Python file to adapt to the new regulatory formatting.

---

## 🛑 VC Challenge & Investor Critique
**The Self-Healing Security Nightmare:** Allowing an LLM to dynamically rewrite Python infrastructure code (`SecEdgarXBRLParserTool.py`) and hot-reload it in production is a massive security and stability vulnerability. An attacker or a simple hallucination could inject arbitrary code, infinite loops, or malicious exfiltration logic directly into the production environment. No institutional investor would underwrite this risk.

### Critical User Journeys (CUJs)
1. **The Code Hallucination:** The SEC updates an XBRL tag. The LLM tries to "heal" the parser but hallucinates the `BeautifulSoup` syntax, causing the tool to silently drop half the balance sheet. The Agent proceeds with flawed data.
2. **The Execution Sandbox Breach:** The agent writes code that accidentally imports and runs `os.system('rm -rf /')` or leaks the Alpaca API keys while trying to "debug" the parser.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Immutable Tooling**: LLMs are never permitted to alter `.py` files in production. The "Self-Healing" concept is entirely redefined.
2. **Fallback Cascades**: If the strict `SecEdgarXBRLParserTool` fails (returns `None` due to a tag change), the Flow automatically catches the exception. Instead of rewriting code, it falls back to a **Semantic RAG Tool**. It downloads the raw SEC filing, embeds it in a local ChromaDB, and uses a prompt to ask "Extract TotalAssets." 
3. **Alerting over Mutating**: The system sends an async alert to the human developer: "XBRL Parser failed on Ticker X. Using expensive RAG fallback. Please patch the Python tool." This ensures uptime while keeping the codebase perfectly secure.
