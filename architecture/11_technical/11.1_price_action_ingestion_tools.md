# Price Action Ingestion Tools

## High-Frequency Granularity
The Technical Analyst requires clean, unbroken price arrays to compute short-term micro-capital entry points.

## Professional Implementation
- **Tool**: `OHLCVStreamerTool`
- **Logic**: Connects to the Alpaca Market Data API. Instead of pulling daily candles, it defaults to 1-minute and 5-minute aggregations, forcing the data into Pandas DataFrames before handing it off to the CrewAI execution context. This prevents the LLM from trying to "read" thousands of raw JSON lines.

## Evolutionary Mechanism
If the Technical Analyst frequently complains that 5-minute data is too slow for sudden breakouts, the Meta-Review Crew autonomously edits the `OHLCVStreamerTool` configuration, updating the default timeframe from `5Min` to `1Min` for specific high-beta ticker symbols.

## 🛑 VC Challenge & Investor Critique
**The Context Window Blowout:** VCs know that LLMs process data sequentially via tokens. Feeding a 1-minute OHLCV DataFrame for a full trading day (390 rows) directly into an LLM's context window is computationally suicidal. Even if you "prevent it from reading raw JSON," converting 390 rows of Pandas data to a Markdown table will consume ~15,000 tokens per ticker. Doing this repeatedly will hit rate limits instantly and cost hundreds of dollars a week in API fees.

### Critical User Journeys (CUJs)
1. **The Context Overflow:** The Technical Agent requests 1-minute data for the past 5 days to analyze a trend. The tool dumps 1,950 rows into the prompt. The LLM hits its `max_tokens` limit, truncates the most recent (and most important) data, and hallucinates a trade based on data from 3 days ago.
2. **The Formatting Hallucination:** The LLM misaligns the Markdown table rows in its "attention" mechanism, confusing the 'High' column with the 'Volume' column, and executes a trade because it thought the price reached 5,000,000.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Strict Data Hiding**: The LLM must **never** see raw OHLCV arrays, ever. The `OHLCVStreamerTool` does not return tables to the LLM. It saves the Pandas DataFrame to an internal SQLite/Parquet file. 
2. **Abstracted Querying**: The Technical Agent is only given tools like `Get_Trend_Direction(ticker, timeframe)` or `Check_Support_Level(ticker)`. The Python tool calculates these metrics locally on the hidden DataFrame and returns a simple string to the LLM: `"The 5-minute trend is strongly bullish. Nearest support is $150.25."` This drops token usage from 15,000 per query to 15.
