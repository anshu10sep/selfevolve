# Indicator Calculation Delegation

## Offloading Math from the LLM
Large Language Models are notoriously bad at complex sequential math. They should not be calculating a 14-period RSI manually.

## Professional Implementation
- **Delegation Strategy**: The Technical Analyst Agent does not compute indicators. It uses `TaLibCalculationTool`.
- **Mechanism**: The agent simply passes `{"ticker": "AAPL", "indicator": "RSI", "period": 14}` to the tool. The tool runs the Python TA-Lib library and returns the exact float value `32.4`. The LLM only interprets the result.

## Evolutionary Mechanism
If the agent relies heavily on MACD but realizes its win rate is poor during sideways chop, the agent uses a `BacktestIndicatorTool` during its self-reflection phase. It discovers that Bollinger Bands perform better in chop, and permanently updates its internal system prompt to request Bollinger Bands instead of MACD under low-volatility conditions.

---

## 🛑 VC Challenge & Investor Critique
**The Sequential Latency Nightmare:** If the agent is allowed to query indicators sequentially via a tool, the latency stacks. If the strategy requires RSI, MACD, and Bollinger Bands to make a decision, the agent calls the tool (2s round trip), reads the RSI, thinks (3s), calls the tool for MACD (2s), thinks (3s), calls the tool for BBs (2s). Total time: 12+ seconds. In a volatile market, the setup is gone.

### Critical User Journeys (CUJs)
1. **The Sequential Slippage:** The agent is checking 5 indicators. By the time it finishes the 5th tool call, the price has moved 1%, invalidating the first 4 indicators. The agent is trading on staggered, desynced data.
2. **The Invalid Parameter Hallucination:** The agent decides it wants a "Custom Exponential Moving Average," hallucinates the parameter `{"indicator": "CEMA", "period": 14.5}`, and the `TaLibCalculationTool` crashes because TA-Lib doesn't support floats for periods or fake indicators.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Pre-Computation & Batching**: Agents are stripped of the ability to request indicators one by one. The `Data Ingestion Node` running before the CrewAI Flow pre-computes a standard "Feature Vector" (RSI, MACD, BB, ATR, EMA-9/21/200) for all watchlist tickers simultaneously using vectorized Pandas (0.01 seconds).
2. **Vector Injection**: The pre-computed JSON block `{"RSI_14": 32.4, "MACD_hist": -0.5, "BB_width": 2.1}` is injected directly into the Technical Agent's context at instantiation. The agent simply reads the state; no tools, zero latency stacking, no invalid parameter crashes.
