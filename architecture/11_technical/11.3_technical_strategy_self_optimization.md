# Technical Strategy Self-Optimization

## Dynamic Entry Parameters
Static technical rules (e.g., "Buy when RSI < 30") decay over time as market microstructure evolves.

## Professional Implementation
- **Dynamic Thresholding**: The Technical Agent maintains its active threshold values in a database table, not hardcoded in its prompt.
- **Context Injection**: During instantiation, it pulls `{"current_rsi_oversold_threshold": 28.5}`.

## Evolutionary Mechanism
Every weekend, a dedicated `Optimization Crew` reruns the past month's data. It uses grid search to test varying RSI thresholds. If it determines that an RSI of 26 yielded a 15% higher win rate over the last 30 days than an RSI of 30, it overwrites the database. On Monday, the Technical Agent automatically begins executing using the optimized threshold of 26.

---

## 🛑 VC Challenge & Investor Critique
**The Peak Curve-Fitting Trap:** Any quantitative VC will immediately flag grid-searching optimal parameters over a 30-day window as the absolute definition of curve-fitting. If the last 30 days were a steady bull market, the system will optimize the RSI threshold to buy every tiny dip (e.g., RSI=40). When the market inevitably switches to a bear regime on day 31, the over-optimized RSI=40 trigger will catch a falling knife over and over, liquidating the account.

### Critical User Journeys (CUJs)
1. **The Over-Fitted Slaughter:** The system optimizes the MACD crossover threshold perfectly for the low-volatility summer months. In September, volatility spikes 300%. The optimized threshold triggers 50 false signals in one day. *Result: Complete drawdown due to regime-blind optimization.*
2. **The LLM Grid Search Implosion:** Asking an LLM (`Optimization Crew`) to conduct a mathematical grid search over a massive dataset is computationally impossible. It will hallucinate the "optimal" parameter based on semantic bias rather than actual math.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Banish LLM Grid Search**: Optimization must be purely quantitative. We implement a Python `Walk-Forward Optimization Service` (using `scikit-learn` or `backtrader`). The LLM is strictly banned from choosing the numerical parameters.
2. **Regime-Specific Parameter Matrices**: We do not maintain one "optimized" value. The database maintains a matrix: `[RSI_Oversold_Bull: 40, RSI_Oversold_Bear: 20, RSI_Oversold_Chop: 30]`. 
3. **Flow Routing**: At runtime, the `Macro Agent` classifies the current regime (Bull/Bear/Chop). The CrewAI Flow then selects the pre-optimized parameter specifically tied to that regime. This prevents a parameter optimized for a bull market from executing during a bear market.
