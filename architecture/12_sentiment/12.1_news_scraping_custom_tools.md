# News Scraping Custom Tools

## Filtering the Noise
The Sentiment Analyst must separate actionable financial news from clickbait to accurately judge market momentum.

## Professional Implementation
- **Tool Suite**: `BloombergRSSParserTool`, `BenzingaAPIWrapper`, `TwitterCashtagScraperTool`.
- **Filtering Layer**: Before data hits the Sentiment Agent, an ultra-fast `gpt-4o-mini` triage node scores the headlines for relevance. If relevance is low, the headline is dropped.

## Evolutionary Mechanism
If the agent triggers trades based on news sources that are consistently late (i.e., the price already moved before the article dropped), the `Meta-Review Crew` identifies the latency delay. It automatically blacklists that specific RSS feed URL from the `BloombergRSSParserTool` configuration, evolving its data diet to focus only on zero-latency sources.

## 🛑 VC Challenge & Investor Critique
**The Execution Squeeze & API Rate Limits:** "Twitter Cashtag Scraping" in 2026 is brutally expensive due to X's enterprise API pricing, and free scraping tools are routinely IP-banned. If the system relies on scraping Twitter to determine sentiment for a $100 account, the data costs will exceed the capital base in a week. Furthermore, `gpt-4o-mini` triage node adds ~500ms of latency per headline. During a major news drop (e.g., FOMC), 500 headlines will drop simultaneously, bottlenecking the triage node and missing the alpha.

### Critical User Journeys (CUJs)
1. **The Cost Overrun:** The Twitter API wrapper gets caught in a loop querying `$AAPL` during an Apple product launch, racking up a $50 API bill in 10 minutes.
2. **The Triage Congestion:** 200 headlines hit the RSS feed. The `gpt-4o-mini` triage node processes them sequentially. By the time it identifies the critical "CEO Resigns" headline at position #150, the stock has already halted trading. *Result: Complete failure of the news ingestion pipeline due to sequential LLM queuing.*

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Drop Expensive Social APIs for Phase 1/2**: X/Twitter and premium Bloomberg APIs are stripped from the micro-capital architecture. We rely exclusively on free, high-signal push feeds (e.g., Alpaca News WebSocket, SEC Edgar RSS).
2. **Vectorized NLP over LLM Triage**: Replace the `gpt-4o-mini` triage node with a local, open-source Transformer model (e.g., `FinBERT`). FinBERT can process 500 headlines in 50 milliseconds locally (zero API cost, zero network latency). Only headlines that FinBERT scores with >0.8 sentiment magnitude are passed to the actual CrewAI Sentiment Agent for deep reasoning.
