# Sentiment Weighting Evolution

## Balancing Sentiment vs Technicals
Sentiment shouldn't always outweigh hard technical support levels, especially in a $100 portfolio where a sudden drop is fatal.

## Professional Implementation
- **Dynamic Weighting Engine**: The `Portfolio Manager Agent` uses a dynamic weight variable to blend the Sentiment Analyst's output with the Technical Analyst's output. (e.g., `SentimentWeight = 0.3`, `TechWeight = 0.7`).

## Evolutionary Mechanism
If the system experiences a series of losses during "meme stock" rallies by ignoring sentiment and relying purely on overbought technicals, the `Meta-Review Crew` identifies this regime change. It autonomously increases the `SentimentWeight` variable in the database to `0.6` for highly volatile tickers, evolving the entire Crew's decision-making hierarchy to respect momentum over technicals in that specific environment.

---

## 🛑 VC Challenge & Investor Critique
**The Whipsaw Evolution Danger:** Dynamically mutating the core weights of the portfolio's decision engine based on a "series of losses" is incredibly dangerous. If the system loses 3 trades in a row during a meme-stock rally, evolves `SentimentWeight` to 0.6, and the next day the meme-stock rally crashes and normal technical trading resumes, the system is now heavily weighted in the wrong direction. This guarantees you will always be one step behind the market, buying the top and shorting the bottom.

### Critical User Journeys (CUJs)
1. **The Lagging Regime Shift:** Market transitions from irrational exuberance to a technical bear market on Monday. The system loses money Monday and Tuesday. On Wednesday, it updates its weights. By Thursday, the market transitions to a sideways chop. The system's weights are perfectly optimized for Monday's market, ensuring failure on Thursday.
2. **The LLM Weighting Hallucination:** The `Meta-Review Crew` (an LLM) is asked to calculate the new optimal `SentimentWeight`. It hallucinates the math and sets the weight to `9.0` instead of `0.9`. The Python execution layer crashes due to array normalization errors.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **No LLM Math Mutations**: The `Meta-Review Crew` is explicitly banned from editing numerical weight variables. 
2. **Kalman Filtering / Python Ensembles**: The `Dynamic Weighting Engine` is rewritten as a deterministic Python layer. We implement a mathematical filter (e.g., an Exponential Moving Average of agent success rates). If the Technical Agent's predictions have been accurate 80% of the time over the last 100 trades, its weight naturally drifts to `0.8` mathematically. The LLM does not decide the weights; the mathematical success rate of the agents' past predictions dictates their future voting power automatically.
