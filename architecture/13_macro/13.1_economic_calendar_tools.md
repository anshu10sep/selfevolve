# Economic Calendar Tools

## Mapping Systemic Catalysts
Trading through a Federal Reserve rate decision without knowing it's happening is a quick way to liquidate a micro-capital account.

## Professional Implementation
- **Tool**: `MacroCalendarScraperTool`
- **Functionality**: Pulls the daily/weekly economic calendar (e.g., from ForexFactory or TradingEconomics APIs) and filters for "High Impact" events like CPI, NFP, and FOMC.
- **Crew Integration**: The `Macro Analyst Agent` runs this tool at 08:00 AM daily, producing a "Blackout Window" schedule.

## Evolutionary Mechanism
If the system executes a trade 5 minutes before an unflagged mid-tier economic event (like JOLTS Job Openings) that causes massive slippage, the Macro Agent reviews the log. It autonomously updates the `MacroCalendarScraperTool` to classify "JOLTS" as a "High Impact" event going forward, permanently expanding its systemic risk awareness.

## 🛑 VC Challenge & Investor Critique
**The Silent Tool Failure:** Free or cheap macroeconomic calendar APIs are notorious for changing their DOM structure or dropping data points. If the `MacroCalendarScraperTool` fails to parse the ForexFactory HTML table properly, it returns an empty list. The CrewAI agent assumes "No events today" and proceeds to trade right through a surprise FOMC rate hike. VCs will demand extreme fault-tolerance for systemic risk guards.

### Critical User Journeys (CUJs)
1. **The Null Return Trap:** The calendar API returns HTTP 500. The Python tool catches the error and gracefully returns `[]` to the agent. The agent logs "Safe to trade" and the portfolio is liquidated by a massive CPI print.
2. **The Timezone Hallucination:** The agent pulls an event scheduled for "14:00 GMT". The LLM tries to convert this to EST in its "head" and calculates "10:00 AM EST" instead of "09:00 AM EST". The blackout window is applied to the wrong hour. *Result: The system trades directly into the volatility spike.*

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Fail-Safe Blackouts**: If the `MacroCalendarScraperTool` fails (HTTP 500 or parsing error), it must **not** return an empty list. It must throw a `CriticalMacroException`. The CrewAI Flow catches this and immediately sets `GLOBAL_BLACKOUT = TRUE` for the entire day. The system assumes maximum risk if it cannot prove safety.
2. **Deterministic Timezone Normalization**: The LLM is strictly forbidden from dealing with timezones. The Python tool normalizes all event times to UNIX UTC timestamps before returning the JSON. The `Blackout Window` is enforced by the deterministic execution router, ensuring the agent cannot hallucinate the schedule.
