# Portfolio Constraint Validation

## The Final Checkpoint
Even if the Judge approves a trade, it must be mathematically validated against the current T+1 ledger.

## Professional Implementation
- **Tool**: `ConstraintValidatorTool`
- **Mechanism**: The Judge must pass its Pydantic `ExecutionSignal` through this tool before finalizing the Crew task. The tool checks if `fractional_notional_value` > `current_settled_cash`. If true, it raises a hard error, forcing the Judge to downsize the trade.

## Evolutionary Mechanism
If the Judge frequently proposes trades that exceed settled cash (e.g., trying to buy $20 when only $5 is settled), it indicates a failure in context awareness. The Meta-Review Crew autonomously edits the Judge's core prompt to prominently display: "CRITICAL: You only have $X settled," injecting the variable dynamically, forcing it to respect the constraint before writing the schema.

## 🛑 VC Challenge & Investor Critique
**The Downsizing Slippage Risk:** A VC will note that forcing an LLM to "downsize the trade" in real-time introduces massive execution risk. If the LLM proposes a $20 trade based on a specific Kelly Criterion calculation, and the tool forces it to "try again," the LLM will take 5-10 seconds to generate a new $5 trade. During those 10 seconds, the breakout happens. Furthermore, if you downsize a $20 position to $5, the risk/reward math might no longer justify the API token cost of the trade.

### Critical User Journeys (CUJs)
1. **The Context Window Freeze:** The Judge proposes a $20 trade. Tool rejects it. Judge proposes $15. Tool rejects it. Judge proposes $10. Market closes. *Result: Wasted compute, missed execution.*
2. **The Minimum Notional Trap:** Alpaca has minimum notional limits for certain fractional shares (e.g., $1.00 minimum). The LLM downsizes the trade to $0.50. The tool accepts it, but Alpaca rejects the final HTTP request.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Deterministic Auto-Downsizing**: The LLM must *never* be asked to recalculate math. The CrewAI Flow passes the Judge's `ExecutionSignal` to the `ConstraintValidatorNode`. If `$20 > current_settled_cash ($5)`, the Python node *silently* and instantaneously overwrites the `fractional_notional_value` to `$5.00` and submits the order to Alpaca. Zero latency delay, zero LLM retry costs.
2. **Expected Value Sanity Check**: After auto-downsizing to $5.00, the Python node runs a check: `if notional_value < 5.00: abort_trade()`. This guarantees that the system does not place micro-trades where the fractional slippage and API token costs outweigh the potential profit.
