# Reflexion Framework and Self-Evolution

## The Requirement for Continuous Evolution
Traditional algorithmic systems update by retraining neural network weights—a slow, prohibitive process unsuitable for intraday evolution. The system utilizes the **Reflexion framework**, a "System 2" cognitive process that improves decision-making through linguistic and verbal reinforcement.

## The Reflexion Loop
1. **Outcome Retrieval**: After a trade closes, the Trading-Decision Agent queries the Alpaca API for actual realized PnL, execution latency, and bid-ask slippage.
2. **Prediction Comparison**: It retrieves the original predictive thesis from the prior Debate Workflow.
3. **Objective Labeling**: Compares actual market outcomes against historical predictions, labeling the action as profitable or loss-making.
4. **Linguistic Post-Mortem**: Drafts a detailed textual reflection distilling reasons for success/failure (e.g., "Loss incurred by buying breakout during low-volume lunchtime hours").

## Permanent Behavioral Modifiers
These reflections are codified and stored in the Agent Workflow Memory. In future sessions, when encountering analogous conditions, the reflection is injected into the context window. This alters the agent's style preferences, rewriting its operational logic and continuously evolving trading efficacy without direct human coding.
