# Retrieval-Augmented Generation for Agents

## Contextualizing Market History
LLMs only know data up to their training cutoff. They need RAG to understand recent market regimes.

## Professional Implementation
- **Vector Database**: A local Qdrant or Milvus instance stores chunks of daily market summaries and the system's own daily PnL journals.
- **RAG Tool**: `HistoricalRegimeQueryTool`. Agents use semantic search (e.g., "What happened the last time CPI came in 0.2% hotter than expected?") to retrieve exact historical analogues from the vector DB.

## Evolutionary Mechanism
The database grows daily. As the system trades, it writes its own experiences into the RAG database. Over months, the agents evolve from relying on generic LLM financial knowledge to relying on their own proprietary, heavily vectorized trading journal, creating a truly unique and specialized intelligence.

## 🛑 VC Challenge & Investor Critique
**The Semantic Mismatch in Finance:** A VC quant will highlight that semantic search (cosine similarity on text embeddings) is notoriously bad at financial time-series correlation. If an agent searches for "CPI hotter than expected," the vector database will retrieve documents where the words "CPI" and "hot" appear together. It might retrieve a document from 2021 where CPI was 7% and a document from 2024 where CPI was 3%. The market reactions to those two events are entirely different, but the text embeddings look mathematically identical.

### Critical User Journeys (CUJs)
1. **The False Analogue:** The agent retrieves a journal entry from 2022: "CPI hot, market crashed, we shorted and won." It applies this logic in 2026. However, in 2026, the Fed has already signaled rate cuts, so a hot CPI causes a rally. *Result: The semantic RAG mechanism provides historically irrelevant context, causing a massive loss.*
2. **The Vector Pollution:** After 6 months, the database contains 150 entries about "FOMC meetings." The semantic search retrieves the top 5, effectively randomizing which historical meetings the agent bases its logic on.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Multi-Modal Retrieval (Metadata Filtering)**: We must augment pure vector search with hard metadata filtering. When saving to Qdrant, the payload must include strict JSON metadata: `{"asset_class": "equities", "fed_funds_rate": 5.25, "vix_level": 15}`. 
2. **The Python Query Router**: The agent is NOT allowed to do raw semantic searches. The agent calls a Python tool `Get_Historical_Analogue(event="CPI")`. The Python script queries the broker API for the *current* Fed Funds Rate and VIX, then constructs a Qdrant query that forces an exact metadata match *before* doing the semantic vector search. This guarantees the LLM only receives historical context from structurally identical macroeconomic regimes.
