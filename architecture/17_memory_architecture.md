# Agentic Memory Architecture

## Memory as the Foundation of Evolution
For the Reflexion framework to function, evolutionary post-mortems must be stored and retrieved with extreme precision. The infrastructure relies on a scalable memory architecture designed for multi-agent workflows, utilizing **Mem0** and **Elasticsearch**.

## Hierarchical Memory Layers

### 1. Workspace Memory (Short-Term)
The immediate cognitive scratchpad. Retains the current session's analysis, pending T+1 settlement clearance times, unresolved market blockers, and intraday volatility spikes. Fluid and frequently flushed.

### 2. User Memory (Durable Constraints)
Stores immutable facts. Maintains the continuous, real-time balance of the portfolio, overall user risk profile, and specific operational directives that must not be violated.

### 3. Global / Agent Workflow Memory (AWM)
The paramount layer for self-evolution. Acts as a vast database storing successful "task recipes," historical trading strategies, and the linguistic post-mortems generated by Reflexion.

## Multi-Signal Retrieval
Retrieval uses parallel scoring passes combining:
- Semantic vector embeddings
- Precise keyword matching
- Entity matching

This ensures that if current market structures mirror historical failures, the specific failure reflection is injected directly into the Bear Agent's prompt.
