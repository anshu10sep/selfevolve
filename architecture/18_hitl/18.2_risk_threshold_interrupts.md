# Risk Threshold Interrupts

## Defining the Kill-Switches
The system must know exactly when it is "out of bounds" and requires human intervention.

## Professional Implementation
- **The Overwatch Daemon**: An external Python thread running alongside the CrewAI process. It continuously polls the Alpaca account equity.
- **Hard Interrupts**:
  - Daily Drawdown > 5% of starting equity.
  - API Error Rate > 20% in the last hour.
  - VIX spikes > 15% intraday.

## Evolutionary Mechanism
If the Overwatch Daemon halts the system, the human operator must restart it. When restarting, the operator passes an `override_reason`. The system's `Portfolio Manager Agent` ingests this reason, updating its internal risk map. Over time, it learns the exact mathematical boundaries of the operator's true risk tolerance, evolving its internal interrupt triggers to match.

## 🛑 VC Challenge & Investor Critique
**The Polling Latency Disconnect:** A VC architect will immediately challenge the phrase "continuously polls." If the Overwatch Daemon polls the Alpaca account equity every 60 seconds, a massive market crash can wipe out 30% of the $100 micro-capital account in 15 seconds before the daemon even notices the 5% threshold was breached. Polling is fundamentally insufficient for hard kill-switches.

### Critical User Journeys (CUJs)
1. **The Polling Gap Wipeout:** At 10:00:00, equity is $100. At 10:00:15, a flash crash hits an open position, dropping equity to $85. At 10:00:59, the position drops to $70. At 10:01:00, the daemon polls, sees $70, and triggers the 5% kill-switch far too late.
2. **The Silent Failure:** The Overwatch Daemon crashes due to a random Python dependency error. The main CrewAI process is entirely unaware its guard is down and continues trading normally, completely unprotected.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **WebSocket Equity Streaming**: The Overwatch Daemon must be ripped off REST API polling. It must connect to the Alpaca `trade_updates` and `account_updates` WebSocket streams. This provides sub-millisecond, push-based awareness of account equity changes, allowing the Kill-Switch to fire the instant the 5% threshold is breached.
2. **The "Dead Man's Switch"**: We must implement a heartbeat mechanism. The Overwatch Daemon must write a timestamp to Redis every 1 second. The CrewAI Execution Flow checks this Redis key before *every single trade*. `if (current_time - redis_heartbeat) > 3_seconds: ABORT_TRADE`. This guarantees the LLM physically cannot execute a trade if the risk-management daemon has crashed.
