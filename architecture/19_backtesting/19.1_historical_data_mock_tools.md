# Historical Data Mock Tools

## Simulating the Past
To backtest a CrewAI system, the agents cannot realize they are in the past; they must believe it is live.

## Professional Implementation
- **Tool Overrides**: `AlpacaMarketDataTool` and `BloombergRSSParserTool` are swapped out using Dependency Injection.
- **The Time-Machine Wrapper**: A custom test-harness that feeds the Crew data strictly from a frozen historical date (e.g., `2024-01-15T10:00:00Z`). If the Technical Agent asks for "current price," the mock tool returns the price at that exact historical second.

## Evolutionary Mechanism
This isolated environment allows the `Meta-Review Crew` to run thousands of simulated days. The system autonomously evolves its prompts, backstories, and indicator thresholds entirely in the dark, compressing years of evolutionary trial-and-error into a multi-day compute sprint before ever touching live money.

## 🛑 VC Challenge & Investor Critique
**The Lookahead Bias Guarantee:** A quantitative VC will tell you that it is virtually impossible to build a "Time-Machine Wrapper" for an LLM without introducing lookahead bias. The LLM's foundational model was trained on data up to 2025/2026. If you feed it a frozen historical date of `2024-01-15`, the LLM *already natively knows* what happened in the market on `2024-01-16`. It will "predict" the future flawlessly, making the entire backtesting evolution loop fundamentally fraudulent.

### Critical User Journeys (CUJs)
1. **The Foundational Leak:** The time-machine feeds the agent data from 2022. The agent is asked to evaluate FTX. Because the LLM was trained in 2025, it "knows" FTX is a fraud, even though the 2022 mock data says it's healthy. It shorts it, generating a fake massive profit in the backtest.
2. **The Impossible Sprint:** Running "thousands of simulated days" using multi-agent CrewAI orchestration requires millions of API calls. At $0.01 per call, running a 3-year backtest (750 trading days, 10 loops a day) will cost $75.00+ in API fees, instantly blowing the budget of a $100 micro-capital project before it goes live.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Banish LLM Backtesting**: We completely abandon the idea of running the LLM through historical time-machines for strategy optimization. It is computationally ruinous and structurally biased.
2. **Deterministic Component Testing Only**: Backtesting is strictly isolated to the **Deterministic Execution Layer** (Python rules, indicators, risk management limits). We use traditional backtesting libraries (`VectorBT` or `Backtrader`) to validate the mathematical parameters. 
3. **Forward-Paper-Testing Only**: For the LLM Intelligence Layer, we rely 100% on real-time, forward-walking **Paper Trading**. There is no synthetic history. The system must prove its evolutionary merit on live, out-of-sample data, which guarantees zero foundational lookahead bias.
