# Backtest Evolution Metrics

## Proving Expected Value (EV)
The goal of Phase 1 is not just to see if the agents can converse, but to mathematically prove positive EV.

## Professional Implementation
- **Metric Dashboards**: Tracking Sharpe Ratio, Max Drawdown, Win Rate, and Average Risk/Reward across the synthetic runs.
- **Evolution Checkpoints**: The backtest is run in batches (e.g., 30-day windows). After each window, the `Meta-Review Crew` runs its prompt-evolution cycle.

## Evolutionary Mechanism
By comparing the Sharpe Ratio of Window 1 (pre-evolution) against Window 2 (post-evolution), developers can mathematically verify if the AI's autonomous self-editing is actually improving the strategy or causing catastrophic overfitting. Only when the evolution curve shows sustained, non-overfit improvement does the system clear Phase 1.

## 🛑 VC Challenge & Investor Critique
**The Sharpe Ratio Manipulation:** A quant VC will point out that Sharpe Ratio is easily manipulated in backtests. If the system evolves to execute hundreds of tiny, high-probability scalps (taking 0.1% profit) and holds losers indefinitely (never realizing the loss), it will generate an astronomically high synthetic Sharpe Ratio with a smooth upward equity curve—until it hits a margin call or a 100% wipeout.

### Critical User Journeys (CUJs)
1. **The Martingale Evolution:** The system evolves a Martingale strategy: every time it loses, it doubles the next bet. In a limited 30-day window, this works flawlessly, generating a perfect 4.0 Sharpe Ratio. The developer approves the evolution. On day 31, a 5-day losing streak liquidates the account.
2. **The Overfitting Illusion:** Window 1 Sharpe is 0.5. The system evolves. Window 2 Sharpe is 1.5. However, Window 2 was just a massive bull market that lifted all assets. The system didn't get smarter; it just got lucky with the macro timing.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Sortino & Calmar Over Sharpe**: We replace Sharpe Ratio with the **Sortino Ratio** (penalizes only downside volatility) and the **Calmar Ratio** (Return relative to Maximum Drawdown). This mathematically prevents the system from evolving "hide the loss" or Martingale strategies, as the Calmar ratio will instantly collapse if an unrealized drawdown gets too deep.
2. **Cross-Regime Validation Checkpoints**: "Window 1 vs Window 2" is insufficient. Before any evolved prompt is approved by the pipeline, it must be run through 3 mathematically distinct synthetic periods: a Bull run, a Bear crash, and a Sideways chop. The evolution is only approved if the EV is positive across the *aggregate* of all three adversarial environments.
