# Sandbox Environment Configuration

## The Transition to Real-Time
Phase 2 removes the historical time-machine and exposes the system to live market data streams, but routes orders to Alpaca's Paper API.

## Professional Implementation
- **Environment Variables**: `.env` configurations are explicitly mapped to `ALPACA_PAPER_KEY` and `ALPACA_PAPER_SECRET`.
- **System Flag**: A global variable `IS_PAPER_TRADING = True` is injected into the Manager's `UserContext`.

## Evolutionary Mechanism
The system must learn to handle real-time latency. In Phase 1, data retrieval was instant. In Phase 2, a WebSocket might drop. The `Data Ingestion Agents` track these timeouts. If a WebSocket disconnects, the agent autonomously edits its own Python error-handling logic to implement a 3-second exponential backoff and reconnect sequence, evolving its network resilience in real-time.

## 🛑 VC Challenge & Investor Critique
**The Sandbox Delta:** A VC operating a trading desk knows that "Paper API" and "Live API" are never 100% identical. Alpaca's Paper API often features perfect fills, zero rate-limit stringency compared to live, and occasionally stale Level 1 data. Optimizing an LLM system against a Paper API risks building a model that assumes perfect execution geometry, which will shatter upon contact with live retail flow.

### Critical User Journeys (CUJs)
1. **The Perfect Fill Bias:** The system consistently places limit orders tightly inside the spread. On the Paper API, Alpaca's simulator fills them immediately. The system evolves to rely on this. In Live, those orders sit pending indefinitely because market makers ignore them.
2. **The Global Flag Failure:** The `IS_PAPER_TRADING` flag is a single point of failure. A developer accidentally deploys with `IS_PAPER_TRADING = False` while still using test logic. The system executes highly experimental evolutions with real $100 capital.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Architectural Hard-Segregation**: A simple boolean flag (`IS_PAPER_TRADING`) is unacceptable for production infrastructure. We must enforce environment segregation at the CI/CD level. The `Production` Docker container physically cannot load the `Paper` API keys because they are injected via AWS Secrets Manager based on the VPC environment, preventing accidental cross-contamination.
2. **Simulated Penalty Injection**: To counter the "Perfect Fill Bias" of the Paper API, the `AlpacaOrderSubmissionTool` is wrapped in a `FrictionNode`. Even if Alpaca Paper reports a successful instant fill, the `FrictionNode` artificially delays the success response to the CrewAI flow by a randomized 200-800ms, and subtracts 0.1% from the fill price. This forces the LLM to evolve its strategies in a hostile, pessimistic environment, ensuring readiness for Phase 3 Live.
