# Paper Trading Performance Review Crew

## The Dress Rehearsal
Phase 2 must prove that the entire system—APIs, LLM token costs, and evolved prompts—is economically viable.

## Professional Implementation
- **The Evaluator Crew**: A separate, isolated CrewAI instance that audits the Main Trading Crew at the end of the Phase 2 sprint.
- **Cost vs PnL**: It calculates total API token costs (OpenAI/Anthropic fees) subtracted from the paper PnL.

## Evolutionary Mechanism
If the system generated $5 in paper profit but consumed $7 in API tokens, the strategy is a net negative. The Evaluator Crew forces a massive evolution event: it drastically truncates the backstories of all sub-agents, disables the Sentiment Analyst, and restricts trading to 1-hour windows to cut token costs by 60%, forcing the system to evolve into a leaner, economically viable organism before live deployment.

---

## 🛑 VC Challenge & Investor Critique
**The Over-Correction Amputation:** An investor will recognize that if you have an engine generating gross profit ($5), the alpha is real. If the Evaluator Crew unilaterally "amputates" the system (disabling the Sentiment Analyst, truncating backstories) to save $2 in API fees, it might destroy the very intelligence that generated the $5 profit in the first place. This is cutting off your leg to save money on shoes. You evolve a cheap system that makes $0 instead of an expensive system that makes $5.

### Critical User Journeys (CUJs)
1. **The Lobotomized Deployment:** The Evaluator Crew panics over token costs. It truncates the Risk Manager's prompt to save 500 tokens. The next day, the Risk Manager forgets how to calculate position sizing and blows up the paper account.
2. **The False Economy:** Disabling the Sentiment Analyst saves $1.00 per day. However, three days later, an Elon Musk tweet crashes the market. Without the Sentiment Analyst, the system buys the dip and loses $15.00.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Model Distillation over Amputation**: The system must never delete intelligence or disable agents to save money. Instead, the architecture must support **Autonomous Model Swapping (Routing)**. If a specific agent's task is proven to be repetitive and highly accurate, the system routes that specific task to a cheaper model (e.g., from GPT-4o to Llama-3-8B via Groq) while keeping the full, verbose prompt intact.
2. **SLA-Based Evolution**: The Evaluator Crew is replaced by a deterministic Python `FinOpsNode`. It evaluates the ROI of each individual agent. If an agent's token cost exceeds its historical alpha contribution by 20%, it is not disabled; it is simply rate-limited (e.g., allowed to run only twice a day instead of 10 times). This preserves the intelligence layer while enforcing strict cost Service Level Agreements (SLAs).
