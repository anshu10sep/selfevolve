# Production API Key Management

## Protecting the $100
Live deployment requires strict separation of sandbox and production credentials. Hardcoded keys are an unacceptable risk.

## Professional Implementation
- **Environment Hydration**: The CrewAI system expects `ALPACA_API_KEY` and `ALPACA_SECRET_KEY` in the environment.
- **Bootloader**: A custom Python bootloader fetches these keys securely from a HashiCorp Vault or AWS Secrets Manager instance at runtime, hydrates the `os.environ` namespace, and immediately deletes the local trace.

## Evolutionary Mechanism
If an API key is accidentally exposed or nears its 90-day rotation policy, the `Compliance Officer Agent` triggers an autonomous Key Rotation Protocol. It requests a new key pair via the Alpaca OAuth/Admin API, updates the secure Vault, and hot-reloads the Crew environment without dropping active WebSocket connections.
