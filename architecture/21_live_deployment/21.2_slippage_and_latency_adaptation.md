# Slippage and Latency Adaptation

## The Cost of the Spread
In a live environment, market orders rarely fill at the exact quoted price. This slippage can destroy micro-capital margins.

## Professional Implementation
- **Slippage Accounting Tool**: After every trade, a `FillReconciliationTool` compares the intended `limit_price` with the actual `fill_price` from the Alpaca execution receipt.
- **Dynamic Spread Buffers**: The Manager Agent is forced to add a `+0.5%` slippage buffer to its theoretical PnL calculations before approving any setup.

## Evolutionary Mechanism
If the system trades a highly liquid asset (e.g., $SPY) and notices the slippage is consistently `0.01%`, the Meta-Review Crew autonomously shrinks the generic `0.5%` buffer constraint for $SPY specifically. This evolves the system to take trades on $SPY that it might previously have rejected for insufficient margin, optimizing asset-specific profitability.
