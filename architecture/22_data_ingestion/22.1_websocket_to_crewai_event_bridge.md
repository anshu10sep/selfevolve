# WebSocket to CrewAI Event Bridge

## Connecting Real-Time to Conversational AI
CrewAI agents cannot natively "listen" to a continuous WebSocket stream. They are task-based.

## Professional Implementation
- **The Event Bridge**: A separate Python asynchronous `asyncio` process maintains the Alpaca WebSocket connection.
- **Trigger Logic**: The Bridge monitors the stream for specific events (e.g., a volume spike > 500% of the 5-minute average). When an event hits, the Bridge instantiates a new CrewAI task, injecting the specific anomalous data directly into the Manager's `Task.description`.

## Evolutionary Mechanism
The Event Bridge logic starts simple (e.g., "Trigger on 5% price drops"). If this results in 50 useless CrewAI instantiations that drain API tokens without finding trades, the Manager Agent feeds back a `BridgeConfigUpdate`. The Bridge autonomously rewrites its own filter logic to "Trigger on 5% price drops ONLY IF VIX > 20", evolving the ingestion layer to be more selective.
