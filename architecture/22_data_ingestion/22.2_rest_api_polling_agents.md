# REST API Polling Agents

## Handling Asynchronous Endpoints
Not all data (like SEC filings or macroeconomic releases) is available via WebSockets.

## Professional Implementation
- **Dedicated Polling Crew**: A background Crew scheduled via `cron` or `APScheduler`.
- **Intelligent Polling**: Instead of blindly polling the SEC EDGAR API every minute, the Polling Agent only polls aggressively during expected release windows (e.g., the exact day a 10-Q is historically due).

## Evolutionary Mechanism
If a company changes its earnings release from Pre-Market to After-Hours, the agent's morning polling wastes tokens. The `Fundamental Analyst` detects the "No New Filing" response, cross-references recent PR wire changes, and updates the Polling Agent's `cron` schedule string in the database. The system autonomously evolves its data-fetching schedule to match reality.
