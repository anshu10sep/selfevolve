# Feature Engineering Nodes

## Preparing Data for the AI
Raw price data is difficult for an LLM to interpret semantically.

## Professional Implementation
- **The Pre-Processing Flow**: Before the data reaches the CrewAI context window, a Python node engineer features. It calculates RSI, MACD, and Bollinger Bands.
- **Semantic Translation**: Crucially, it translates these numbers into text. Instead of passing `RSI: 25`, it passes `RSI: 25 (Highly Oversold)`.

## Evolutionary Mechanism
If the system correlates a string of losses with "False Breakouts," the Meta-Review Crew updates the Feature Engineering Python file. It instructs the node to add a new feature: `Volume_Anomaly_Score`. It writes a script to compare the current 5-minute volume to the 30-day average, injecting `Volume: 300% Above Average (High Conviction Breakout)` into the LLM's context.

## 🛑 VC Challenge & Investor Critique
**The Semantic Hallucination Injection:** VCs will correctly identify that passing "Semantic Translations" like `(Highly Oversold)` directly to an LLM introduces massive confirmation bias. If you inject the words "High Conviction Breakout" into an LLM's prompt, the LLM will almost certainly output a "BUY" signal, completely bypassing its own analytical reasoning. You are essentially pre-biasing the LLM, making the entire CrewAI architecture an expensive rubber stamp for your Python feature node.

### Critical User Journeys (CUJs)
1. **The Semantic Trap:** The Python node sees volume spike and injects `(High Conviction Breakout)`. The Technical Agent reads this. Even though the price action is actually a bearish rejection wick, the agent is biased by the words "High Conviction" and buys into the top.
2. **The Feature Engineering Bug:** The Meta-Review Crew autonomously edits the Python file to add the `Volume_Anomaly_Score`. It introduces a `DivideByZero` error when a stock is halted (Volume = 0). The entire pre-processing flow crashes, blinding the AI.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Pure Quantitative Injection**: Do not translate numbers into semantic judgments for the LLM. The LLM must do its own reasoning. Pass strict JSON: `{"RSI": 25, "Vol_Anomaly": 3.0}`. The prompt must instruct the LLM: "Synthesize these metrics. You define what is oversold based on the macro context."
2. **Immutable Feature Nodes**: As established, the LLM is permanently banned from editing Python code. If the Meta-Review Crew wants a new feature (like Volume Anomaly), it must submit a `FeatureRequest` to the human developer. The developer writes the Python, tests it, and deploys it. The AI only consumes the features; it never engineers the code.
