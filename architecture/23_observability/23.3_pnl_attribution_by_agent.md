# PnL Attribution by Agent

## Identifying the Alpha Source
In a multi-agent system, when a trade makes money, you must know *which* agent provided the edge.

## Professional Implementation
- **Telemetry Tagging**: Every executed trade in the PostgreSQL database is tagged with the exact `Debate_Log`. 
- **Attribution Logic**: If the Technical Agent scored the trade a 9, but the Fundamental Agent scored it a 2, and the trade makes a 5% profit, the database attributes +1 "Alpha Point" to the Technical Agent and -1 to the Fundamental Agent.

## Evolutionary Mechanism
The Orchestrator reviews the rolling 30-day "Alpha Points" leaderboard. If the Macro Analyst Agent's score drops below -10, the system deduces the agent is actively harming the portfolio. It autonomously demotes the Macro Agent, reducing its weighting in the Judge's synthesis formula to near zero, starving the underperforming agent of influence.

## 🛑 VC Challenge & Investor Critique
**The Confounding Variable Problem:** Attribution is the hardest problem in quantitative finance. If the Technical Agent scores a trade a 9 because of a MACD crossover, and the trade makes 5% because the Federal Reserve unexpectedly cut interest rates 10 minutes later, attributing that profit to the Technical Agent is fundamentally false. The Technical Agent was wrong, it just got lucky. If the system evolves its weights based on this false attribution, it will over-leverage technical indicators right before they fail.

### Critical User Journeys (CUJs)
1. **The False Positive Promotion:** The Sentiment Agent hallucinates a bullish thesis. The stock rallies because of an unpredicted short-squeeze. The Sentiment Agent gets an "Alpha Point" and its weight is increased. The next day, it hallucinates again, buys, and loses 20% because there is no short-squeeze to save it.
2. **The Defensive Penalty:** The Risk Management Agent constantly votes "Reject" on high-risk, high-reward trades. The system takes one anyway and it wins big. The Risk Agent gets a negative Alpha Point. Eventually, the Risk Agent is demoted to zero influence, completely removing all safety guardrails from the portfolio.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Asymmetric Risk Attribution**: The system must never penalize an agent for recommending "Reject" on a trade that ultimately wins. Capital preservation is the priority. "Alpha Points" can only be adjusted based on the outcome of *accepted* proposals.
2. **Isolating Variables via A/B Testing**: To truly attribute PnL, you cannot rely on post-trade synthesis. The system must run **Shadow Portfolios**. When the Technical Agent says "Buy" and the Fundamental says "Reject", the Live portfolio might follow the Judge and Buy. Simultaneously, a Paper portfolio executes the *opposite* (Reject). The system tracks the divergence between the portfolios over hundreds of trades to isolate the true Expected Value of each individual agent's logic.
