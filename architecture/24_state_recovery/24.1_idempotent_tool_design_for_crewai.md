# Idempotent Tool Design for CrewAI

## Preventing Duplicate Executions
In an LLM loop, an agent might accidentally call the `AlpacaOrderSubmissionTool` twice if it gets confused by a network timeout.

## Professional Implementation
- **Idempotency Keys**: The Manager Agent generates a unique UUID for every proposed trade `Intent`.
- **Tool Enforcement**: The `AlpacaOrderSubmissionTool` requires this UUID. Before hitting Alpaca, it checks the local SQLite database. If the UUID exists as `FILLED` or `PENDING`, the tool intercepts the call and returns: "Error: Trade already executed."

## Evolutionary Mechanism
If an agent repeatedly attempts duplicate executions (e.g., getting stuck in a loop), the system counts the `Idempotency_Blocks`. The Meta-Review Crew detects the looping behavior, analyzes the agent's chain-of-thought trace, and autonomously edits the agent's `Thought -> Action` sequence prompt to force a `TaskComplete` signal immediately after receiving a successful tool receipt.
