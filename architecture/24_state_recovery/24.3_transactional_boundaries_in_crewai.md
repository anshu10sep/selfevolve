# Transactional Boundaries in CrewAI

## Atomic Operations
A trade setup is only valid if both the Buy order and the associated Stop-Loss/Take-Profit orders are placed.

## Professional Implementation
- **Bracket Orders**: The `Judge Agent` does not place three separate orders. It is strictly constrained to use the `BracketOrderSubmissionTool`.
- **Atomicity**: This tool maps to Alpaca's native bracket order API. If the server crashes one second after the HTTP POST, Alpaca guarantees the stop-loss is attached to the primary buy order.

## Evolutionary Mechanism
If the system detects that bracket orders are failing to fill because the take-profit leg is too tight (causing Alpaca to reject the order structure), the `Meta-Review Crew` identifies the `OrderReject` error code. It autonomously rewrites the constraint equations in the Judge Agent's prompt, widening the minimum allowable spread between the entry and take-profit parameters for future brackets.
