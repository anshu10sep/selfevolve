# API Key Rotation Policies

## Securing the Keys to the Kingdom
If the Alpaca API keys or OpenAI keys are leaked, the $100 account (or the massive AWS billing account) will be drained in minutes.

## Professional Implementation
- **Secret Management**: Keys are never stored in `.env` files in production. They are injected at runtime via AWS Secrets Manager.
- **Rotation**: A Lambda function automatically rotates the Alpaca API keys every 30 days, pushing the new keys to Secrets Manager and triggering an ECS container restart.

## Evolutionary Mechanism
If the `Meta-Review Crew` detects repeated `HTTP 401 Unauthorized` errors from the OpenAI API, it deduces that the key has been compromised or revoked. It autonomously triggers an emergency Lambda function to revoke the current key, generate a new one via the OpenAI administration API, and restart the orchestration sequence, minimizing downtime from security breaches.

---

## 🛑 VC Challenge & Investor Critique
**The Autonomous Key Generation Nightmare:** Allowing an LLM (the `Meta-Review Crew`) to autonomously generate new API keys and inject them into production infrastructure is a catastrophic security risk. If the system enters an error loop, it might generate 1,000 new OpenAI API keys in an hour, triggering massive billing fraud alerts and getting your entire organization banned from OpenAI. Furthermore, Alpaca API keys grant access to real financial capital; they must be guarded by strict IAM policies, not AI agents.

### Critical User Journeys (CUJs)
1. **The API Key Spam:** The OpenAI API goes down for maintenance (returning 503s, which the LLM misinterprets as 401s). The AI panics, assumes compromise, and generates a new API key every 5 seconds for an hour.
2. **The Privilege Escalation:** To allow the AI to rotate its own OpenAI keys, you must grant the AWS ECS container Administrator-level access to the OpenAI organization API. A prompt injection attack on the LLM could instruct it to extract those admin credentials and send them to an external server.

## 🛠️ Refinement & Architecture Update
To make this robust:
1. **Zero-Trust AI Infrastructure**: The CrewAI Orchestrator (the LLM) is strictly granted **Least Privilege**. It only has access to the *usage* keys. It has absolutely zero access to administrative APIs, IAM roles, or Secrets Manager logic.
2. **Deterministic Infrastructure Automation**: API key rotation is handled entirely outside the AI's purview. A standard AWS EventBridge cron job triggers a Python Lambda function (written by a human) to rotate keys. The AI is simply restarted by ECS and fed the new keys via environment variables. The AI is fundamentally unaware that rotation is even occurring.
