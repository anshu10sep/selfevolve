# Regulatory Compliance Checkers

## Pre-Execution Scrubbing
The system must never execute a trade on an asset that violates international sanctions or internal restricted lists.

## Professional Implementation
- **Tool**: `RestrictedListScrubberTool`
- **Execution Block**: The `Judge Agent` must pass the proposed ticker through this tool. The tool cross-references OFAC sanctions lists, FINRA halt lists, and the `UserContext` "Do Not Trade" list.
- **Action**: Raises a hard block if a match is found.

## Evolutionary Mechanism
The `Compliance Officer Agent` runs a daily `cron` task to pull updated OFAC and FINRA XML feeds. It autonomously updates the local restricted database. If the AI originally had a highly profitable strategy trading Russian ETFs, and a new sanction drops, the system autonomously blocklists the tickers overnight, evolving its trading universe to remain strictly compliant without human intervention.
