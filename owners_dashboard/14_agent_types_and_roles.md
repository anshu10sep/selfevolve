# Agent Types and Roles

The Jarvis ecosystem relies on specialized agents rather than a single monolithic model to accomplish complex tasks efficiently.

## Core Agent Types

### 1. Planner Agent
- **Role**: Breaks down large epics into smaller, manageable tickets or jobs.
- **Tools**: Access to Jira/Linear APIs, Markdown parsers.

### 2. Coder Agent
- **Role**: Writes the actual code.
- **Tools**: Read/write access to the cloned repository, LSP (Language Server Protocol) integration, shell access in an isolated sandbox.

### 3. Reviewer Agent
- **Role**: Reviews PRs generated by Coder Agents.
- **Tools**: Access to static analysis tools, linters, GitHub PR review API.

### 4. QA / Testing Agent
- **Role**: Writes and executes unit and integration tests.
- **Tools**: Access to test frameworks (PyTest, Jest), browser automation (Playwright/Puppeteer).

### 5. DevOps Agent
- **Role**: Handles deployments, modifies CI/CD pipelines, provisions infrastructure.
- **Tools**: Terraform CLI, AWS/GCP APIs, Docker.

## Dashboard Visualization
The Owner's Dashboard groups active agents by their Type. This allows the owner to quickly answer questions like, "Do we have a bottleneck because there are 10 Coder Agents but only 1 Reviewer Agent?"
