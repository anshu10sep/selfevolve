# Feedback Loop Mechanism

For a self-evolving system to work, it must have a mechanism to receive and incorporate feedback.

## Types of Feedback

### 1. Human (Owner) Feedback
- The owner reviews a PR generated by Jarvis via the dashboard.
- If the owner rejects the PR, they leave a comment.
- The Integration Service parses the comment, converts it into an Event, and routes it back to the Jarvis Core.
- Jarvis updates its internal heuristics: "Avoid using pattern X in this repository."

### 2. Automated System Feedback
- An agent writes code, but the CI/CD pipeline fails.
- The Watchdog detects the failure, pulls the CI logs, and feeds them back to the agent.
- The agent learns from the stack trace and attempts a fix.

## Dashboard Visualization
The "Evolution & Learning" section of the dashboard tracks this feedback loop.
- Displays a metric: `Feedback Incorporation Rate` (How often Jarvis successfully fixes an issue based on human feedback).
- Allows the owner to manually input feedback directly into the Jarvis Core Model's memory bank via a text box on the UI.
